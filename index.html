01 — Background

Why run locally?

For a clinician-researcher, local LLM deployment offers a specific and compelling combination of advantages that cloud APIs cannot replicate: patient and research data never transits external servers, usage is unconstrained by subscription tiers or rate limits, and models can be fine-tuned on domain-specific corpora without surrendering that data to a third party.

The open-source landscape has matured substantially. As of March 2026, well-chosen local models reach approximately 80–90% of frontier proprietary performance on everyday tasks. The gap is most pronounced in multi-step reasoning chains and creative synthesis — but for literature review, structured analysis, clinical correspondence, and research coding in R or Python, the delta is often imperceptible in practice.

Scope of this assessment

All models listed here run on a single consumer or prosumer GPU costing under approximately $4,500. Multi-GPU enterprise configurations — required for the full-scale GLM-4.7, DeepSeek V3.2 671B, Kimi K2.5, and Mistral Large 3 — have been deliberately excluded. All models are available via Ollama or LM Studio without complex orchestration.

02 — Terminology

Hugging Face & Ollama are not models

A common point of confusion: Hugging Face and Ollama are distribution and deployment platforms, not model families. Every model in this report is, in the precise sense, a Hugging Face model — the weights are hosted there. And every model is deployable via Ollama, which was the recommended runtime throughout.

Hugging Face

Repository / hosting platform

Analogous to GitHub for machine learning. All major open-weight models publish their weights here. You download from Hugging Face; you do not run models "on" Hugging Face unless using their Inference API.

Ollama

Local inference runtime

Handles model download, quantization selection, GPU offloading, and serves an OpenAI-compatible API locally. Three terminal commands suffice: install, pull, run. Does not develop its own models.

LM Studio

GUI-based runtime (macOS / Windows)

A graphical front-end for running GGUF models locally. Preferred for users who want a chat interface without configuring a terminal. Pulls models from Hugging Face directly.

llama.cpp / vLLM

Inference backends

Lower-level inference engines. llama.cpp powers Ollama under the hood. vLLM is preferred for production multi-user deployments with higher throughput requirements.

The relationship is linear: model creators (Alibaba, Microsoft, Google, DeepSeek, Meta, OpenAI) publish weights to Hugging Face, which are then fetched and served by Ollama or LM Studio on your local hardware.

03 — Infrastructure

Hardware requirements

VRAM is the hard constraint. All figures below assume Q4_K_M quantization, which compresses model weights to 4-bit precision and reduces memory requirements by approximately 75% relative to FP16, with benchmark degradation typically under 3 points on general tasks. Performance degrades significantly when models are forced to offload layers to system RAM — a drop from 40 tokens/sec to 8 tokens/sec has been measured on the same model when only partial VRAM fit is achieved.

08 — Scope decisions

Models excluded for individual deployment

The following models were assessed in the full enterprise survey but excluded here on the basis of hardware, infrastructure, or licensing constraints incompatible with individual operation.

GLM-4.7 / GLM-5
Requires 200 GB+ VRAM across multiple H100 GPUs. No realistic path for individual deployment. The full model is excluded; no consumer-accessible distilled variant exists as of this assessment.
Kimi K2.5
1 trillion parameter MoE architecture; multi-GPU requirement. MIT license has a 100M monthly active user commercial threshold, introducing ambiguity for any use beyond strictly personal.
DeepSeek V3.2 (671B full)
Requires 350 GB+ VRAM at 4-bit quantization. The 32B distilled variant (DeepSeek R1 32B) is retained as it is specifically optimized for single-GPU deployment.
GPT-OSS 120B
Requires a single H100 80 GB at minimum — approximately $25,000 in hardware. The 20B variant is retained and is individually deployable on a 24 GB consumer card.
Mistral Large 3 (675B full)
Full MoE model requires multi-GPU infrastructure. Mistral's smaller models (7B) are individually deployable but do not outperform Qwen3 and Phi-4 at equivalent VRAM, and are therefore not listed as primary recommendations.
Llama 3.3 70B
Requires 48 GB+ VRAM for Q4 deployment. Partial CPU offloading reduces speed to impractical levels on 24 GB hardware. Qwen3 32B and DeepSeek R1 32B deliver comparable or superior performance at half the VRAM cost.
09 — Recommendation

Deployment guidance

For a 24 GB GPU workstation (RTX 4090 / 3090)

Run both DeepSeek R1 32B and Qwen3 32B via Ollama simultaneously, switching between them by task type. DeepSeek R1 for structured analytical work — literature analysis, differential reasoning, R or Python debugging — where its explicit chain-of-thought output is an advantage. Qwen3 32B for drafting, correspondence, summarization, and instruction-following tasks where fluency matters more than auditability.

QUICK START

ollama pull deepseek-r1:32b && ollama pull qwen3:32b — approximately 40 GB combined disk space. Both models load in under 60 seconds on NVMe storage.

For Apple Silicon (M4 Max 64 GB or 128 GB)

The unified memory architecture makes this platform uniquely capable for individual use. A 64 GB M4 Max can run DeepSeek R1 32B or Qwen3 32B at full FP16 precision — a higher quality than Q4 quantization achievable on a 24 GB NVIDIA card. The 128 GB variant can accommodate Qwen3 72B at Q4, approaching A-tier enterprise performance on consumer hardware. Ollama on macOS uses Metal acceleration natively with no additional configuration.

For a 16 GB GPU workstation

Phi-4 14B is the primary recommendation for reasoning and analytical tasks. Supplement with Gemma 3 12B when long-context document ingestion is required (128K vs. Phi-4's 16K). Both fit in 16 GB simultaneously if system RAM is sufficient for context management.

Domain-specific fine-tuned alternatives

Several medically fine-tuned derivatives of the base models listed here are worth evaluating for clinical applications — notably Meditron (fine-tuned from Llama on medical literature), OpenBioLLM (biomedical domain adaptation), and BioMedLM. These are hosted on Hugging Face and loadable via Ollama. They typically sacrifice some general capability for stronger performance on clinical terminology, guideline recall, and medical reasoning. A separate assessment of these domain-specific variants is available on request.

Summary recommendation

For a clinician-researcher with a private practice: an RTX 4090 workstation running DeepSeek R1 32B and Qwen3 32B via Ollama provides the optimal balance of capability, privacy, and individual deployability as of March 2026. Total hardware investment is approximately $2,000–$2,500 for the GPU plus a mid-range workstation. All patient and research data remains local. No subscription costs. No rate limits.

10 — Practical examples

Reference hardware builds for DeepSeek R1 32B

The research workstation (premium)

Primary GPU: NVIDIA RTX 4090 24 GB | CPU: Intel i9-14900K or AMD Ryzen 9 9950X | System RAM: 64–128 GB DDR5 | Storage: 2 TB NVMe SSD | Power: 1200W 80+ Gold PSU

This configuration allows you to run DeepSeek R1 32B at full Q4 quantization (approximately 20 GB VRAM) with headroom for system processes and browser instances. The CPU is fast enough to handle tokenization without GPU blocking. Estimated total cost: $3,500–$4,200.

The budget workstation (solid)

Primary GPU: NVIDIA RTX 4080 Super 16 GB | CPU: Intel i7-14700 or AMD Ryzen 7 7700 | System RAM: 32 GB DDR5 | Storage: 1 TB NVMe SSD | Power: 850W 80+ Gold PSU

DeepSeek R1 32B will run at Q4 quantization with approximately 1–2 GB of margin. This is the minimum recommended tier for comfortable deployment. At lower quantization levels (Q3), you gain margin for simultaneous workloads, but sacrifice reasoning quality. Estimated total cost: $2,200–$2,800.

The Mac alternative (unified memory)

System: Apple MacBook Pro M4 Max 128 GB | Accelerator: Metal (native, no GPU purchase required) | Storage: 1 TB SSD

The M4 Max's unified memory architecture eliminates the traditional VRAM bottleneck. DeepSeek R1 32B loads entirely into the memory pool and runs at native FP16 precision — a quality tier higher than Q4 quantization on NVIDIA cards with the same inference speed. The 128 GB variant is overkill for a single 32B model, but provides headroom for RAG systems and multi-model workflows. Estimated cost: $6,500 (laptop); favorable for clinicians who need portability.

Installation & usage

All three configurations deploy identically via Ollama:

INSTALLATION

curl -fsSL https://ollama.ai/install.sh | sh
ollama pull deepseek-r1:32b

Once the model is pulled, interact with it via a CLI chat interface, integrate it into your application via the HTTP API (localhost:11434), or use third-party UI tools like Open WebUI or Lollms.

Performance expectations

Tokens per second: 40–80 tokens/s on a 4090; 25–45 tokens/s on a 4080 Super; 50–90 tokens/s on M4 Max (metal acceleration). Context window: 128K tokens (standard for DeepSeek R1 32B). Cost per query: $0.00 (all local; your electricity cost only).

11 — Build cost analysis

DeepSeek R1 32B deployment: Three build paths compared

This section provides a detailed cost breakdown and capability comparison for three distinct approaches to deploying DeepSeek R1 32B locally: two custom self-builds optimized for cost and capability, and one pre-built system offering convenience at a premium. All pricing verified March 18, 2026.

The builds at a glance

⚠ Price correction — DDR5 RAM crisis: DDR5 RAM prices have surged over 400% since mid-2025 due to an AI-driven NAND supply shortage. A 64 GB DDR5-6000 kit that cost ~$200 in mid-2025 now runs $750–$900. All totals reflect verified March 2026 pricing. Earlier cost estimates significantly understated RAM costs.

Build A: Single GPU workstation (most economical)

GPU: 1× RTX 3090 24 GB (used) · CPU: Ryzen 7 7700X · RAM: 64 GB DDR5-6000 · Total cost: ~$2,515–$2,865

This is the lowest-cost entry point for DeepSeek R1 32B deployment. A single used RTX 3090 provides 24 GB VRAM, sufficient for Q4 quantization of the 32B model. The platform (AM5 + DDR5) allows future GPU upgrades without CPU/mobo replacement. Expected token throughput: 18–28 tokens/second.

  • ✓ Best VRAM-per-dollar efficiency (~$38–$42 / GB)
  • ✓ Comfortable performance for single-user clinical/research work
  • ✓ Future upgrade path: swap GPU for RTX 4090 or newer without platform change
  • ✗ Used GPU carries no warranty; requires careful vetting
  • ✗ Cannot run 70B-class models (insufficient VRAM)
  • ✗ Self-assembly required; moderate technical complexity

Build B: Dual GPU workstation (best capability per dollar)

GPU: 2× RTX 3090 24 GB (used) with NVLink bridge · CPU: Ryzen 7 7700X · RAM: 64 GB DDR5-6000 · Total cost: ~$3,455–$4,025

For ~$800–$1,100 more than Build A, you acquire 48 GB total VRAM (NVLink-connected), enabling deployment of 70B-parameter models at Q4 quantization and full-precision inference on 32B models. This is the strongest choice for researchers who anticipate experimentation beyond R1 32B. Expected token throughput: 25–38 tokens/second.

  • ✓ 48 GB VRAM pool supports 70B models and full-precision 32B inference
  • ✓ Comparable VRAM-per-dollar to Build A (~$36–$42 / GB)
  • ✓ Faster inference than Build A (25–38 vs. 18–28 tok/s)
  • ✗ Higher sourcing risk: two used GPUs to vet from different sellers
  • ✗ Complex assembly: NVLink bridge configuration, larger PSU/case required
  • ✗ No warranty on either card

Build C: Alienware Area-51 prebuilt (convenience and warranty)

GPU: RTX 5090 32 GB GDDR7 (new) · CPU: Intel Core Ultra 9 285K (24-core) · RAM: Stock 32 GB + ~$300 aftermarket upgrade to 64 GB · Total cost: ~$5,299 (including RAM upgrade)

The RTX 5090 is a generation ahead (Blackwell architecture, 2025) and delivers approximately 2× the token throughput of either 3090-based build. This is currently the only practical retail avenue to acquire an RTX 5090 near MSRP. The prebuilt premium over Build A is now approximately $2,400–$2,800 when RAM costs are accurately accounted for. Expected token throughput: 45–65 tokens/second.

  • ✓ Newest GPU architecture; 2× faster inference than Build A
  • ✓ RTX 5090 access: currently unavailable in retail; prebuilt is the only avenue
  • ✓ Full 1-year system warranty from Dell; zero assembly required
  • ✓ Stock DDR5 is cheaper to upgrade than buying custom
  • ✗ Highest total cost (~$2,400 premium over Build A)
  • ✗ Poor VRAM-per-dollar ratio (~$166 / GB vs. ~$38–$42 for custom builds)

Where to source components

GPU (used RTX 3090): eBay ($800–$1,000 with verified seller ratings), Reddit r/hardwareswap ($750–$950, trader flair), or Facebook Marketplace ($700–$900 with local pickup inspection).

CPU (Ryzen 7 7700X): Micro Center ($229.99, in-store), Amazon ($245–$265, watch for price dips via CamelCamelCamel alerts), or Newegg (especially CPU + motherboard bundles, which often discount by $30–$60).

RAM (64 GB DDR5-6000): Volatile pricing — verify immediately before purchase. Amazon ($750–$850, set price alerts) or Newegg bundle deals (often $100–$200 cheaper when bundled with CPU/mobo). G.Skill Flare X5 and Crucial Pro are the most commonly competitive models.

Storage/PSU/Case/Cooler: Amazon for most components; Newegg occasionally offers 5–10% bundle discounts. WD Black SN850X 2 TB NVMe runs ~$200; Corsair RM1000x PSU ~$155; Fractal Define R5 case ~$120; be quiet! Dark Rock 4 cooler ~$75.

NVLink bridge (Build B only): eBay, search "NVIDIA NVLink 3-slot RTX 3090" (~$60–$90). Verify slot pitch and cooler clearance compatibility with your cards before ordering.

Alienware Area-51 (Build C): Dell.com or Alienware.com ($4,999 direct). Purchase the RTX 5090 system directly from Dell for warranty. Buy a separate 2×32 GB DDR5-5600 kit on Amazon (~$300) rather than through Dell's configure-to-order (which typically overcharges 40–60% for memory upgrades).

Final recommendation

Which build?

Build A if your budget is fixed and R1 32B is sufficient for your use case. Build B if you anticipate evaluating 70B-class models or want full-precision FP16 inference (which provides slightly higher quality than Q4 quantization). Build C if you need the fastest possible inference and cannot absorb setup time or hardware sourcing risk — and if you value a warranty on the entire system.
